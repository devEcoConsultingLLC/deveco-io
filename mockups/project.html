Project Overview

This project demonstrates how to deploy a custom trained object detection model on the ESP32-S3 microcontroller. The ESP32-S3 features a dual-core processor running at 240MHz with 512KB of SRAM, making it capable of running lightweight ML models in real-time.

[Project Image: ESP32-S3 with camera module setup]

What You'll Learn

  • Setting up the ESP32-S3 development environment with ESP-IDF
  • Training a custom object detection model using TensorFlow
  • Quantizing the model to INT8 for edge deployment
  • Integrating the camera module with the ML inference pipeline
  • Optimizing memory usage for constrained devices

Hardware Setup

The project uses an ESP32-S3-WROOM-1 module connected to an OV2640 camera module. The camera captures frames at 320x240 resolution which are then preprocessed and fed into the TensorFlow Lite Micro interpreter.

C++ Copy 
// Initialize camera
esp_err_t init_camera() {
    camera_config_t config;
    config.ledc_channel = LEDC_CHANNEL_0;
    config.pixel_format = PIXFORMAT_RGB565;
    config.frame_size = FRAMESIZE_QVGA;
    config.fb_count = 2;
    
    return esp_camera_init(&config);
}

Model Architecture

We use a MobileNetV2-SSD architecture that has been pruned and quantized to fit within the memory constraints of the ESP32-S3. The final model size is approximately 1.2MB with INT8 quantization.
